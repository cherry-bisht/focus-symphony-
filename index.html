The Score

Two ways in — grab a prebuilt binary (recommended, no Go needed) or build from source. Click README above for the full guide.

Quick Start — Prebuilt Binary

  1. Download your platform's binary from the v1.0.0 release ↗
  2. Make executable: chmod +x focus-symphony-linux
  3. Run: ./focus-symphony  or with site-blocking: sudo ./focus-symphony start
  4. Global install: sudo mv focus-symphony-linux /usr/local/bin/focus-symphony
— or build from source (Go required) —
  1. Clone: git clone https://github.com/cherry-bisht/focus-symphony-.git
  2. Build: go build -o focus-symphony main.go
  3. Run: ./focus-symphony start

README.md — focus-symphony v1.0.0

🎵 Focus Symphony

A compiled Go CLI that helps you enter deep focus mode — blocking distracting sites, tuning system processes, and piping ambient audio. All from your terminal.

v1.0.0GoMITArch NativeCross-Platform

What's in v1.0.0

  • Prebuilt binary for Linux (x86_64)
  • Prebuilt binary for macOS (Intel x86_64)
  • Prebuilt binary for Windows (x86_64)
  • Focus session manager — blocks sites via /etc/hosts
  • Built-in terminal music player
  • TUI dashboard for real-time system telemetry
  • No Go installation required for end users

Quick Install — Linux / Arch

wget https://github.com/cherry-bisht/focus-symphony-/releases/download/v1.0.0/focus-symphony-linux chmod +x focus-symphony-linux sudo mv focus-symphony-linux /usr/local/bin/focus-symphony focus-symphony

Quick Install — macOS

wget https://github.com/cherry-bisht/focus-symphony-/releases/download/v1.0.0/focus-symphony-mac chmod +x focus-symphony-mac sudo mv focus-symphony-mac /usr/local/bin/focus-symphony focus-symphony

Quick Install — Windows

Download focus-symphony.exe from the releases page and run directly:

.\focus-symphony.exe

Commands

focus-symphony # launch TUI focus-symphony start # begin focus session (blocks sites — needs sudo) focus-symphony stop # end focus session focus-symphony music # launch terminal music player focus-symphony help # show all commands

Privilege Notes

  • Normal usage (TUI, music, telemetry) — no sudo needed.
  • Site-blocking — requires sudo, modifies /etc/hosts.
sudo focus-symphony start # with site-blocking enabled

Platform Compatibility

  • ✓  Linux — Arch, Ubuntu, Debian, Fedora (native /proc support)
  • ✓  macOS — Intel x86_64
  • ✓  Windows — x86_64
  • ⚠  WSL — partial only. Audio and /etc/hosts behavior differ under WSL.
  • ⚠  Apple Silicon — build from source until arm64 binary is shipped.
⚠ WSL Users: Site-blocking and audio are not fully supported under WSL due to how it handles /etc/hosts and the audio subsystem. Use native Linux for the full experience.
